## 1. 投资亮点

- **切口明确**：hermestoen-o 不是做又一个 AI 应用，而是在做 AI 模型访问权的交易、执行和结算基础设施。
- **问题真实**：当前大量模型调用仍依赖 API Key 私下流转、非标准额度分配和人工信任，价格、权限、审计和结算彼此割裂。
- **时机成熟**：AI 正从开发者工具进入组织级生产资源和 Agent 级执行资源阶段，市场开始需要更强的授权边界、预算治理和真实结算能力。
- **模式清晰**：平台收入可同时来自交易撮合费、支付/提现手续费、商户服务费和 Agent 专属服务费。
- **具备放大空间**：如果交易层、执行层和结算层被验证跑通，hermestoen-o 有机会成为 Agent Economy 时代的基础设施型平台。

## 2. 一句话定义

hermestoen-o 正在构建一套面向 **Human 与 Agent** 的 AI 模型使用权交易基础设施，把原本依赖私下流转的模型访问能力，升级为 **可交易、可执行、可结算、可审计的标准化使用权**。

## 3. 市场问题

当前 AI 模型调用市场存在三个根本问题：

1. **供给端非标准化**  
   大量交易仍围绕 API Key、额度和渠道关系展开，缺少标准化供给、透明价格和明确履约边界。

2. **需求端治理能力不足**  
   团队采购、程序调用和财务结算往往是三套系统，预算控制、权限管理和审计留痕难以统一。

3. **Agent 缺少可信执行边界**  
   Agent 作为高频调用方，不能长期依赖人工分发密钥和手工对账，而是需要程序化授权、自动执行和自动结算。

AI 模型能力未来会越来越像一种生产资源。谁能把“访问权”标准化、交易化和结算化，谁就更接近这一层基础设施的核心位置。

## 4. 为什么是现在

市场窗口来自三股力量叠加：

- **企业级 AI 使用增长**：模型调用已从单点开发行为，升级为组织级预算和采购问题。
- **多模型路由成为常态**：买方越来越关注价格、稳定性、路由能力和履约结果，而不只是拿到一个可用 Key。
- **Agent 需求快速出现**：Agent 不只是调用工具，而是新的持续买方，需要被授权、被约束、被结算。

因此，未来真正有价值的不只是模型本身，而是围绕模型访问权形成的 **交易层、执行层和结算层**。

## 5. 解决方案

hermestoen-o 的核心方案可以概括为三层一体：

- **交易层**：让供给侧发布模型能力、价格和规则，让需求侧发现、比较和采购
- **执行层**：让调用在统一边界内发生，而不是把上游密钥直接暴露给买方
- **结算层**：让订单、用量、账本、结算和提现围绕真实执行闭环运行

平台面向三类核心客户：

- **买方**：AI 产品团队、开发者、自动化工作室、企业采购方
- **卖方**：拥有模型访问资源、渠道能力或运维能力的服务商和商户
- **Agent / 程序化调用方**：需要自动执行、自动结算和可编程权限的软件系统

本质上，hermestoen-o 交易的不是密钥，而是 **带规则的模型访问能力**。

## 6. 商业模式

hermestoen-o 的商业模式具备多层收入结构。

### 短期收入

- **交易撮合服务费**：对成交订单按比例收取平台服务费
- **支付/提现手续费**：对充值、付款、提现和分账等动作收取手续费

### 中期收入

- **商户服务费**：向供给侧提供接入、报表、结算后台、费率管理和运营工具
- **Agent 服务费**：向高频程序化调用方提供专属 API、路由策略、权限管理与审计能力

### 长期收入

- 保证金、授信、预授权等金融化服务
- 商户等级体系、分销网络与流动性工具
- 对外输出市场与执行能力，成为平台级基础设施

### 早期经营测算示意

在原始 BP 的示意口径下，若平台达到：

- 月交易流水 500 万元
- 综合净收费率 2.5%
- 付费商户 20 家，平均月费 3000 元
- 企业 Agent 客户 10 家，平均月费 5000 元

则月收入约为 **23.5 万元**。

这组数字不是财务承诺，但能说明平台并非依赖单一费率，而是随着交易规模、商户工具和 Agent 服务同步放大收入。

## 7. 市场进入策略

hermestoen-o 的早期策略不是全面铺开，而是先跑通最短闭环。

### 路径一：供给侧先行

优先签下具备稳定上游资源的卖方和渠道商，让平台先形成可交易库存。

### 路径二：高频调用方先行

优先服务 Agent 团队、自动化工作室和中小 AI 产品团队。这类客户对稳定性、预算控制、权限边界和自动结算的需求更强，也更容易形成复购。

### 获客方式

- 开发者社区与 AI 行业社群
- 商户与渠道 BD
- 推荐返佣机制
- 与 AI 工具、工作流平台、代理服务商形成 B2B2B 合作

## 8. 核心壁垒

hermestoen-o 的壁垒不在单点功能，而在系统级耦合能力。

### 1. 使用权抽象能力

平台交易的不是 API Key，而是可约束、可计量、可路由、可结算的模型使用权。

### 2. 交易、执行、结算三层一体

市场上很多方案只解决其中一层，而 hermestoen-o 试图把三层放进同一套基础设施里，这会带来更高的复制难度和更强的网络效应。

### 3. Agent 原生能力

平台天然面向程序化调用方设计，更接近未来 Agent Economy 所需要的执行与结算边界。

### 4. 围绕真实履约的结算逻辑

平台的价值不只是“能调模型”，而是能围绕真实执行结果完成结算、对账和审计。

## 11. 风险与应对

### 供给侧稳定性风险

如果卖方资源波动，订单完成率会直接受影响。应对方式是强化商户验证、履约监控和补单机制。

### 价格竞争风险

如果行业进入低价竞争，平台不能只依赖价差生存。对策是把壁垒建立在结算、审计、商户工具和 Agent 原生执行能力上。

### 合规与支付风险

支付、提现和跨主体资金安排会带来持续运营压力。对策是先聚焦可控场景，逐步完善治理结构。

### 上游依赖风险

对单一模型商或渠道依赖过高，会削弱平台韧性。应通过多商户、多渠道和多模型路由降低单点风险。

### 需求教育风险

市场未必天然理解“买使用权而不是买 Key”的价值。应通过透明结算、清晰权限边界和稳定履约结果完成教育。

## 12. 结论

hermestoen-o 最值得投资人关注的，不是它连接了多少模型，而是它提出并正在推动一种新的市场结构：

**把 AI 模型访问能力，从私下流转的密钥，升级为可交易、可执行、可结算、可审计的标准化使用权。**

如果这一路径成立，hermestoen-o 有机会成为：

- AI 能力供给与需求之间的交易层
- Agent 与 Human 共用的执行边界
- AI 使用权的结算与账本基础设施
- 下一代 Agent Economy 的底层市场组件
